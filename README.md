# 40了，得干点儿啥了 — 抖音账号智能运营系统

> 一个数据驱动的 AI Agent 协作系统，用于抖音内容运营的全链路自动化。
>
> **核心理念：用数据闭环替代灵感驱动，让每个内容决策都有据可依。**

## 设计原则

### 1️⃣ 数据驱动，而非灵感驱动
每个内容决策都必须有数据支撑。不凭感觉选题，不看心情写脚本。

**数据链条：**
```
采集 (采集) → 分析 (情报站) → 洞察 (趋势判定) → 选题 (素材筛选) → 脚本 (数据改写) → 发布 → 效果反馈 → 再采集
```

### 2️⃣ 单一数据真相源 (Single Source of Truth)
所有 Agent 读写同一个 `shared/data.json`，不存在多份不一致的数据副本。

```
data.json 结构：
├── materials       素材库（主料+配料+未用角度）
├── account         账号基础信息
├── ranking         作品排名分析
├── creatorCenter   创作者中心深度数据（完播率/2s跳出/粉丝占比）
├── postPublishTracking  发后追踪（D+1/D+3/D+7）
├── selfReview      自检分析记录
└── benchmarking    对标分析（赛道爆款/竞争账号/标题模板）
```

### 3️⃣ 双机隔离 (Dual-Machine Isolation)

| 机器 | 身份 | 操作范围 | 禁用操作 |
|:----|:----|:--------|:--------|
| 🖥️ Windows | 数据采集机 | 对标采集、爆款搜索、数据爬取 | ❌ 登录峰峰账号、发布内容 |
| 🍎 Mac | 账号操作机 | 创作者中心、发布视频、回复评论 | ❌ 采集他人数据、搜同行 |

**为什么这么做：** 防止算法推荐被污染。Windows 搜同行 → 不影响 Mac 上账号的推荐模型。

### 4️⃣ 定时采集 + 异常告警

```
8:00  闹钟 → 采集基础数据（播放/粉丝/作品数变化）
8:30  闹钟 → 创作者中心深度数据（完播率/2s跳出/涨粉）
9:00  闹钟 → 每日简报输出
```

- 不在聊天时主会话的心跳机制检查待办队列
- 安静模式 (23:00-08:00)：只检查事件队列，不做采集
- 异常数据触发微信提醒，正常数据沉默记录

## Agent 协作关系

```
用户/峰峰
  └─ 主 Agent (main) — 统一调度、分发任务、汇总汇报
       ├─ 📡 情报站 (douyin-情报站)
       │    ├─ 对标采集（Windows）
       │    ├─ 创作者中心采集（Mac）
       │    ├─ 数据趋势分析
       │    ├─ 发后追踪
       │    └─ 每日简报
       ├─ 📝 素材管家 (douyin-素材管家)
       │    ├─ 维护素材池 (materials)
       │    ├─ 与峰峰对话挖掘新素材
       │    └─ 标记已用/可用角度
       ├─ 📄 脚本生成器 (douyin-脚本生成器)
       │    ├─ 读数据趋势决定选题方向
       │    ├─ 观点重复检测
       │    └─ 数据驱动改写脚本结构
       ├─ 🎬 标题封面官 (douyin-标题封面官)
       │    ├─ 输出3+套标题方案
       │    ├─ 封面风格统一人设
       │    └─ 标题封面互补不重复
       ├─ 🎥 画面设计师 (douyin-画面设计师)
       │    ├─ 拍摄方案（运镜/机位/光线）
       │    └─ 剪辑点提示
       ├─ 👜 账号管家 (douyin-account-agent)
       │    └─ Manager：读懂峰峰意图、分发事件
       ├─ 🔍 抖音搜索 (douyin-scraper)
       │    └─ 搜索赛道上同行视频
       └─ 📹 视频下载 (douyin-video-fetch)
            └─ 下载参考素材
```

## 数据流详解

### 对标采集流程
```
搜索关键词（Windows）
  → 采集标题/播放/作者/封面特征
  → 结构化写入 data.json → benchmarking
  → Manager 更新对标账号库
  → 标题模板提取
  → 情报趋势判定
```

### 脚本生产流程
```
读 creatorCenter 最新数据
  → 判定「涨」的趋势方向
  → 检查素材池可用素材
  → 观点重复检测（vs最近1条已发视频）
  → 数据驱动改写脚本结构（前10字钩子/结尾类型）
  → 交给标题封面官出方案
  → 交给画面设计师出拍摄方案
```

### 发后追踪流程
```
新视频发布
  → 标记 postPublishTracking
  → D+1：采集初期数据
  → D+3：采集稳定期数据
  → D+7：出最终效果简报
  → 复盘数据反哺脚本生成器
```

## 技术栈

- **Agent 框架：** OpenClaw Gateway (主会话 + subagent)
- **定时任务：** OpenClaw cron (systemEvent闹钟 → 主会话)
- **浏览器自动化：** Playwright (通过 OpenClaw browser tool)
- **双机通信：** OpenClaw node pairing (Windows ↔ Mac)
- **数据存储：** JSON 文件 (shared/data.json)
- **消息推送：** 微信 (异常数据告警)

## 规则集 (Rules of Engagement)

这些规则是通过实战验证的，不是理论推导：

1. **观点重复是最大的雷** — 即使换素材/换场景/换角度，核心观点撞了也不行
2. **前10字必须有钩子** — 陈述句开头=播放低保，悬念/反问/冲突/数字/情绪词开头
3. **结尾4种有效类型** — 没整明白式 / 你发现没式 / 两面选择式 / 自嘲接话式
4. **先查最新数据再写脚本** — 素材不够匹配趋势方向时，通知素材管家要新素材
5. **Windows不走登录，Mac不采同行** — 双机隔离硬边界
6. **子Agent能扛跨机器操作** — isolated session 超时门槛低，走主会话→subagent链路
