# 40了，得干点儿啥了 — 抖音账号智能运营系统

> 一个数据驱动的 AI Agent 协作系统，用于抖音内容运营的全链路自动化。
>
> **核心理念：用数据闭环替代灵感驱动，让每个内容决策都有据可依。**

---

## 什么是「数据驱动」—— 不是口号，是代码

这不是一个"AI帮你写脚本"的工具。这是一个**数据闭环系统**。每一条内容的生产，都始于数据，终于数据，然后回到数据。

具体的做法是——系统每天自动采集**3类数据**，然后**4个决策节点**必须用数据说话。

---

## 核心闭环：3×4 数据驱动模型

### 每天采集哪3类数据？

```
┌─────────────────────────────────────────────────────────┐
│                   每日数据采集                            │
├───────────┬─────────────────────┬───────────────────────┤
│ ① 自账号   │ ② 对标赛道          │ ③ 创作者中心           │
│ 深度数据    │ 爆款分析            │ 官方数据               │
├───────────┼─────────────────────┼───────────────────────┤
│ 播放量     │ 赛道TOP播放标题      │ 完播率 %              │
│ 点赞/评论  │ 封面类型分布         │ 2秒跳出率 %           │
│ 粉丝变化   │ 高频作者识别         │ 粉丝播放占比 %        │
│ 7日对比    │ 标题公式提取         │ 互动率变化            │
│           │                     │ 流量来源分布          │
├───────────┴─────────────────────┴───────────────────────┤
│ 采集方式：Windows对标搜索 + Mac创作者中心 + 双机隔离     │
└─────────────────────────────────────────────────────────┘
```

### 4个数据驱动的决策节点

每个节点有"必须回答的问题"，答不上来就卡住，不往下走。

**节点①：选方向**
```
问题：数据告诉我现在什么在涨？
回答模板：最近7天完播率最高的素材是 X，2秒跳出最低的是 Y，
         粉丝播放占比涨的是 Z，所以这期应该往 ____ 方向写。
```
> 如果素材不匹配趋势 → 通知素材管家要新素材，**不硬写**。

**节点②：定观点**
```
问题：最近1条视频的核心观点是什么？
回答模板：最近一条说的是 ____，核心观点是 ____。
         这期如果写 ____，观点撞了吗？（是/否）
检查：观点重复 → 终止，换素���/换场景都没用，核心观点不能撞。
```

**节点③：写开头**
```
问题：前10字是什么类型？
拒绝：陈述句开头。
允许：反问 / 悬念 / 冲突 / 数字 / 情绪词
检查标准：参考自账号历史TOP5开头的类型分布。
```

**节点④：定结尾**
```
问题：结尾用哪种已验证有效的类型？
允许的4种：
  1. 没整明白式 — "那把镰刀，一开始我就没整明白"
  2. 你发现没式 — "你发现没有，好像跑着跑着就不较劲了"
  3. 两面选择式 — "要么继续撑着，要么换个活法"
  4. 自嘲接话式 — "40岁的人，一个月只有5个粉丝，意味着什么"
拒绝：硬诱导、格言型、抽象大词、总结式。
```

**决策链路图：**
```
数据 → [方向] → 素材匹配？→ [观点] → 观点不撞？→ [开头] → 非陈述句？→ [结尾] → 结尾有效？→ 出脚本
```

## 数据在代码里怎么跑的

### `/shared/data.json` 就是整个系统的核心

```
creatorCenter/
├── lastSnapshot    ← 最新一次采集的快照（包含每个作品的深度数据）
├── snapshots[]     ← 历史快照（可追溯播放变化趋势）
├── lastDelta       ← 本次 vs 上次的对比
├── trend           ← 7日趋势汇总（完播率均值/2秒跳出均值/最佳作品）
└── depthData{}     ← 每个爆款的完整数据分析（流量来源/用户画像/评论热词）

benchmarking/
├── videos[]        ← 赛道对标视频库（34条，含封面类型、色调、标题公式）
├── accounts[]      ← 竞争账号库（14个，含最高播放、赛道定位）
├── visualAnalysis  ← 封面策略分布（跑步侧影+大字占50%等）
└── titleTemplateInsights  ← 已验证的标题公式库（6条）

postPublishTracking/
└── videos[]        ← 每条已发视频的追踪（D+1/D+3/D+7定时采集）
```

### 各Agent怎么用这些数据

| Agent | 读什么数据 | 用来做什么 |
|:------|:---------|:----------|
| 情报站 | 写所有数据 | 采集、分析、写入 |
| 素材管家 | `materials[]` | 知道哪些素材已用、哪些角度可用 |
| 脚本生成器 | `creatorCenter + benchmarking` | 选方向、观点检测、结构优化 |
| 标题封面官 | `benchmarking` 标题公式 + 封面分布 | 出3套不重复方案 |
| 画面设计师 | `benchmarking` 封面特征 | 确定拍摄风格 |

## 设计原则

### 1️⃣ 单一数据真相源 (Single Source of Truth)

所有 Agent 读写同一个 `shared/data.json`。不存在"脚本生成器一份数据，情报站另一份"的情况。

```
Agent A 写 → data.json → Agent B 读
    ↕                    ↕
Agent C 写 ← data.json ← Agent D 读
```

### 2️⃣ 双机隔离 (Dual-Machine Isolation)

| 机器 | 身份 | 操作范围 | 禁用操作 |
|:----|:----|:--------|:--------|
| 🖥️ Windows | 数据采集机 | 对标采集、爆款搜索、数据爬取 | ❌ 登录峰峰账号、发布内容 |
| 🍎 Mac | 账号操作机 | 创作者中心、发布视频、回复评论 | ❌ 采集他人数据、搜同行 |

**为什么：** 防止算法推荐被污染。Windows 搜同行不影响 Mac 账号推荐模型。

### 3️⃣ 定时采集 + 异常告警

```
08:00  闹钟 → 采集基础数据（播放/粉丝/作品数变化）
08:30  闹钟 → 创作者中心深度数据（完播率/2s跳出/涨粉）
09:00  闹钟 → 每日简报输出
```

- 不在聊天时：心跳机制检查待办队列
- 安静模式 (23:00-08:00)：只检查事件，不做浏览器采集
- 数据异常触发微信提醒，正常沉默记录

### 4️⃣ 卡得住，不走捷径

决策节点是**硬性检查**，不是参考建议。观点重复 → 卡住，陈述句开头 → 卡住，结尾类型无效 → 卡住。

宁可今天不产出，也不产出错误的东西。

## 数据流详解

### 对标采集流程
```
搜索关键词（Windows）
  → 采集标题/播放/作者/封面特征
  → 结构化写入 data.json → benchmarking
  → Manager 更新对标账号库
  → 标题模板提取
  → 情报趋势判定
```

### 脚本生产流程
```
读 creatorCenter 最新数据
  → 判定「涨」的趋势方向
  → 检查素材池可用素材
  → 观点重复检测（vs最近1条已发视频）
  → 数据驱动改写脚本结构（前10字钩子/结尾类型）
  → 交给标题封面官出方案
  → 交给画面设计师出拍摄方案
```

### 发后追踪流程
```
新视频发布
  → 标记 postPublishTracking
  → D+1：采集初期数据
  → D+3：采集稳定期数据
  → D+7：出最终效果简报
  → 复盘数据反哺脚本生成器
```

## 技术栈

- **Agent 框架：** OpenClaw Gateway (主会话 + subagent)
- **定时任务：** OpenClaw cron (systemEvent闹钟 → 主会话)
- **浏览器自动化：** Playwright (通过 OpenClaw browser tool)
- **双机通信：** OpenClaw node pairing (Windows ↔ Mac)
- **数据存储：** JSON 文件 (shared/data.json)
- **消息推送：** 微信 (异常数据告警)

## 实战验证的规则

这些不是理论推导，是通过视频数据反复打脸之后才确认的：

1. **观点重复是最大的雷** — 换素材/换场景/换角度都没用，核心观点撞了就是撞了
2. **前10字必须有钩子** — 陈述句开头=播放低保，必须用悬念/反问/冲突/数字/情绪词
3. **结尾4种有效类型** — 没整明白式 / 你发现没式 / 两面选择式 / 自嘲接话式，其他一概禁止
4. **先查最新数据再写脚本** — 素材不匹配趋势时，通知素材管家要新素材，不硬写
5. **Windows不走登录，Mac不采同行** — 双机隔离硬边界
6. **子Agent扛跨机器操作** — isolated session 超时门槛低，走主会话→subagent链路
