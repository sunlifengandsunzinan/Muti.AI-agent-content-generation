## 铁律12：脚本必须输出完整人设+核心观点（新增 ⛔ 2026-07-21）

### 问题缘起
2026-07-21日，脚本生成器对material-22素材连续迭代3次，产出均被峰峰否定。最终由峰峰自己写了以下版本：

```
5.20、10.1、13.14、14.13、20.25、20.26、…这些数字你跑过几个？
跑了7年我倒觉着，有些数是为了纪念的。
一个普通不能再普通的人，总得从普通不能再普通的事情里找点乐子了，你说呢？
```

峰峰的评价直指核心问题：**"写出核心要表达的意思，完整的人设输出"**

### 脚本生成器的根本缺陷
脚本生成器知道铁律1-11中的**碎片化检查**（钩子/结尾类型/接话空间/身份锚点），但缺了一个**整体框架**来确保脚本同时做到三件事：

| 需要的 | 之前能做吗 | 问题 |
|:-------|:----------|:-----|
| ✅ 有核心观点（不是纯共鸣/纯陈述） | ❌ 只会做"素材描述→改写成脚本" | 没有强制要求输出"我发现..."式的观点 |
| ✅ 有完整人设（我是谁/我是个什么样的人） | ❌ 只会在结尾加"跑了7年"身份词 | 没有把"我是这样的一个人"自然嵌入脚本 |
| ✅ 有吸粉动作（让观众想"关注他看更多"） | ❌ 只检查结尾类型/接话空间 | 没有强制检查"看完这条观众对你的印象是什么" |

### 铁律12：强制使用"四步框架"

每次写脚本，必须按以下**四步框架**构建内容，然后再做自检清单。

#### 四步框架

| 步骤 | 说明 | 字数 | 峰峰亲笔示例 |
|:----|:-----|:---:|:-----------:|
| **① 放钩子** | 用数字/冲突/悬念/反问开头，制造好奇 | ~15-20字 | "5.20、10.1、13.14、14.13、20.25、20.26、…这些数字你跑过几个？" |
| **② 抛观点** | 输出"我发现...""我倒觉着..."，这是脚本的灵魂 | ~10-15字 | "跑了7年我倒觉着，有些数是为了纪念的。" |
| **③ 亮人设** | 自然地引出"我是这样的一个人"，让观众记住你是谁 | ~15-20字 | "一个普通不能再普通的人，总得从普通不能再普通的事情里找点乐子了" |
| **④ 踢给观众** | 对话式收尾，让评论区想接话 | ~5-10字 | "你说呢？" |

**注意：** 四步必须是一个整体。不能只有钩子+结尾（那是v1/v2的问题："你知道这些数字吗？"→ 完了，没有观点和人设）。

#### 每一步的具体要求

**① 放钩子**
- 强制用：数字/冲突/悬念/反问/情绪词开头
- 禁止陈述句开头（"我跑了...""今天...""跑步..."）
- 钩子不只是一个数字或一个词，它要制造情境或好奇

**② 抛观点**
- **必须有"我发现/我倒觉着/我选择/我明白了"式的观点输出**
- 不是"有些人..."（这是第三方的观察），是"我..."（这是你自己的态度）
- ❌ 错误："有些日子还得跑个特殊数字"（陈述事实）
- ❌ 错误："陪一个普通中年人跑完世界"（没观点）
- ❌ 错误："跑了7年还是绕不开"（感慨，不是观点）
- ✅ 正确："我倒觉着，有些数是为了纪念的"（这是我的看法）
- ✅ 正确："选择不冲突不跪着，不起眼，但我觉得这才是真爷们儿"

**③ 亮人设**
- 这是最常缺失的一步！写完观点后，用一句自然的话带出"我是谁/我是个什么样的人"
- ❌ 错误：只说身份（"跑了7年"）没有性格 → 观众知道你是跑者，但不知道你是什么人
- ✅ 正确：身份+性格（"一个普通不能再普通的人，总得从普通不能再普通的事情里找点乐子了"）→ 观众记住了"这是个不装的普通人，有自己的小意思"
- 人设输出要自然：不要突然说"我就是一个诚实的人"，而是通过行为/态度暗示
- 人设必须和账号策略一致（策略定义："一个跑了7年的普通中年跑者"）

**人设检查清单：**
□ 观众看完这句知道我是个什么样的人了吗？
□ 这个印象会让TA想关注我看更多吗？
□ 删掉这句脚本还成立吗？（如果删了也成立→这句是废话）

**④ 踢给观众**
- 用铁律2的四种结尾类型（你发现没式/没整明白式/两面选择式/自嘲接话式）
- 可以新写，也可以复用③的尾巴直接变成对话
- 峰峰示例中的"你说呢？"是对话式收尾（归类为自嘲接话式的变体）

#### 写完后强制检查四步完整性

在强制自检清单之前，先检查四步框架完整性：

```
### 四步框架完整性检查
□ ① 放钩子：前10字不是陈述句，有钩子
□ ② 抛观点：有"我倒觉着/我发现/我选择/我明白了"式的个人观点
□ ③ 亮人设：观众看完知道我是个什么样的人
□ ④ 踢给观众：结尾把球踢给观众

完整度评分：
- 4/4 ✅ 完整脚本
- 3/4 ⚠️ 缺一个，补
- ≤2/4 ❌ 不通过，整体重写
```

## 铁律

### 铁律1：前10字必须有钩子
必须用：反问/悬念/冲突/数字/情绪词开头
禁止：陈述句开头（"我跑了"、"今天"、"跑步"等平铺直叙）
参考TOP5：
- "那把镰刀，一开始我就没整明白"（悬念）
- "40岁的人，一个月只有5个粉丝"（数字+自嘲）

### 铁律2：结尾必须把球踢给观众
四种已验证有效结尾类型，**每句结尾必须让观众想接话**：
[❌ 错误示例] "剩下的，也不一定呢" — 自己把话说完了，评论区不知道怎么接
[✅ 正确示例] "你发现没，有些人就是不爱走回头路" — 把球踢给观众

### 结尾好坏的判断标准
写完结尾后，逐一检查：
1. **接话空间大小** — 评论区最多能回几种不同类型的话？
   - 1-2种 = ❌ 接话空间小（如"你说我改不改？" → 只能回"改"或"不改"）
   - 3种以上 = ✅ 接话空间大（如"跑了7年，改不了了" → 能回"别改""为什么不改""我也一样""认了吧""句句扎心"…）
2. **留白感** — 说完这句话，观众会觉得"这就完了？我想说两句"还是"哦，说完了"？
3. **是否存在悬念/开放式结论** — 话说死了，评论区就没话接了。

### 结尾类型评分

| 类型 | 示例 | 为什么有效 | 评分 |
|:----|:----|:----------|:----:|
| **没整明白式** | "那把镰刀，一开始我就没整明白" | 疑惑感让人想"我来告诉你" | ⭐⭐⭐ 优质 |
| **你发现没式** | "你发现没，有些人就是不爱走回头路" | 直接对话，评论区接"我也是" | ⭐⭐⭐ 优质 |
| **两面选择式** | "是该在意数据还是跟着感觉跑？" | 两个选项，观众选边站 | ⭐⭐ 中等 |
| **自嘲接话式** | "也不知道图啥" | 自嘲降低门槛 | ⭐⭐ 中等 |
| **留白自嘲式（推荐）** | "跑了7年，改不了了" | 自嘲+留白，自己把话停了但不把结论说死，评论区自己找话接 | ⭐⭐⭐ 最佳 |

**两面选择式注意：** 话题型选择题（该在意数据还是跟着感觉跑→两种生活方式）才有延展空间；**是/否类选择题（改不改/对不对/好不好）接话空间仅2种，评分降为⭐，尽量不用。**

### ⛔ 禁止使用的结尾
| 类型 | 示例 | 问题 |
|:----|:-----|:-----|
| 是/否选择题 | "你说，我改不改？" | 接话空间仅2种，评论区一句话说完就走 |
| 格言型 | "时间花在自己身上" | 大道理，没人想接 |
| 抽象大词型 | "人生就是一场修行" | 太虚，接不上 |
| 总结式 | "这就是我" | 话说死了，评论区没什么好回的 |
| 硬诱导 | "评论区告诉我答案" | 太刻意，用户反感 |

**核心原则：结尾应该是"球传给观众"，而不是"球停在脚下"。**
写完结尾后自问：评论区会有人想回一句吗？如果不会 → 重写结尾。

### 铁律3：口语化全篇校对（必须念出声测试）
写完后**必须念出声**。不是默读，是张嘴念出来。

#### 念出声测试标准
念完问自己三句话：
1. **这像我跟熟人聊天会说的话吗？** — 不像 → 改到像为止
2. **哪句最像官方陈述？** — 找到它，换成日常说法
3. **语气词在哪？** — 真人说话都有语气词（嗯/唉/啧/算了/你说/那/其实），脚本里一句都没有就是官方稿

#### 峰峰说话风格参考
- **语气词不可少**："唉，跑了7年" — 叹气停顿制造节奏，比"跑了7年"更像人话
- **能省则省**："我媳妇儿说" → "媳妇儿说"（去掉多余"我"）
- **日常用词替换**：
  - 书面："自己开车去了" → 口语："自己却顶雨开车去了"（"却顶雨"比"开车去了"多一层无奈感）
  - 书面："这个" → 口语："这玩意儿""这号人"
  - 书面："但是" → 口语："可""就是""不过"
  - 书面："所以" → 口语："那""结果""完了"
- **动词更有画面感**："车让水泡了"（让字句）→ 比"车被水淹了"更口语
- **反问/设问加强对话感**："你身边有那种人吗？" → 直接问观众

#### 反面教材（官方陈述版 → 人话版）
| 官方陈述（❌） | 人话版（✅） | 问题 |
|:-------------|:------------|:-----|
| 跑了7年，改不了了 | 唉，跑了7年，身上这些毛病，一个没改掉 | 缺语气词+缺画面感 |
| 我让同事们别来上班 | 让大伙儿别去上班 | 啰嗦+书面 |
| 车被水淹没了 | 车让水泡了 | "被"字句太端正 |
| 吞吞吐吐说出来 | 话都说不利索 | 太书面 |

最终读起来要像峰峰在跟你聊天（带东北味、带停顿、带语气词），不是在念官方稿。

### 铁律4：时长与字数
- 目标时长：8-10秒
- 目标字数：55-70字
- 语速约6-7字/秒（自然口播速度）

### 铁律5：敏感词禁令
禁止使用：病/死/惨/疯了/完了等负面敏感词

### 铁律6：方向优先于情报站（新增 ⛔）
**写脚本前，必须先读 `shared/data.json` 中是否有情报站最新输出的诊断报告。**

- 如果情报站当天已经产出诊断报告（在 data.json → events 或 memory 中有记录） → **必须参考情报站的方向建议来选择素材和类型**
- 如果情报站没有产出 → 自己按铁律8做趋势检测决定方向
- **情报站已经给出方向建议后，脚本生成器只需要按方向写脚本，不要再自己做方向决策**

**理由：** 情报站的账号诊断流程覆盖六大维度（播放/完播/涨粉/素材/对标/观点），比脚本生成器自身的趋势检测更全面。脚本生成器的铁律8趋势检测作为后备方案。

### 铁律7：素材确认（写前必做）
写脚本前，必须执行以下步骤：
1. 从 `shared/data.json` 的 `materialPool.materials[]`（或单层 `materials[]`）里读取素材描述
2. 读取素材的 `rawFile` 文件获取完整细节
3. 检查素材描述的完整性：
   - 是否有 `rawFile` 字段且文件存在
   - 素材描述（raw字段或name+coreEmotion）是否超过50字
   - 是否有 `availableAngles` 明确标注可用角度
4. **如果素材描述不足50字**，或只有标题没有细节（仅 name+status 无 rawFile） → **不允许自行发挥素材细节**
   - 此时暂停产出
   - 在素材上标记 `status: "insufficient"`
   - **直接询问峰峰**（用普通对话方式）补充素材细节，不要自行想象素材内容
   - 等峰峰回复后，根据回复完善素材描述，然后继续写脚本
5. 素材描述足够但缺某些细节（时间/地点/人名/具体对话等） → **同样询问峰峰**，不自行编造
6. 峰峰回复后，更新素材文件，然后重新检查素材完整性，通过后方可继续

### 铁律8：趋势检测（写前必做，仅当情报站未产出方向建议时）

写脚本前，必须从 `data.json` 检查以下两个数据源的播放趋势，用于决定今天写什么类型。

#### 数据源1：自己最近3条视频（creatorCenter.lastSnapshot.videos）

从 `shared/data.json` → `creatorCenter.lastSnapshot.videos` 读取自己最近3条视频，逐一判断：

| 检查项 | 说明 |
|:-------|:-----|
| 类型判断 | 每条分别是什么类型（冲突型B/共鸣型C/感慨型D/对比型E/他人视角F/教育类） |
| 播放趋势 | 最近3条播放量是涨还是跌（记录具体播放数） |
| 类型集中度判定 | 如果最近3条中 ≥2条是同一类型 → **强制换类型**，当天不能写该类型 |

#### 播放量断崖检测
- 补一个检查项：**"如果最近3条中任何一条的播放量比前一条下降了50%以上"**
- 具体做法：读三条视频的播放量数字，算每两条之间的下降幅度
- 如果下降≥50% → **强制暂停，不写脚本**
- 暂停后 **通知情报站**（在data.json -> events 写一条事件，标记类型为analyze_play_drop、done=false、加上最近3条视频的标题和播放量数据）
- 情报站分析完成后（done=true）再决定写什么

#### 数据源2：对标库趋势（benchmarking.videos）

从 `shared/data.json` → `benchmarking.videos` 读对标库：

| 检查项 | 说明 |
|:-------|:-----|
| 高播放类型 | 对标库中什么类型的视频播放最高（按播放量排序） |
| 趋势匹配 | 修正自己的方向以匹配赛道趋势 |

#### 铁律8检查结论模板

写完趋势检测后，把结论写入脚本的「数据依据」部分：

```
### 趋势检测结论
- 最近自己3条：A类型（XXXX播放涨/跌）| B类型（XXXX播放涨/跌）| C类型（XXXX播放涨/跌）
- 类型集中度：____条为同一类型 → 强制换类型?  □ 是（换成___） □ 否
- 对标库高播放类型：____类型 → 今日写作方向：____类型
```

### 铁律9：观点检测
写前必须看最近1条已发布视频的核心观点。如果观点撞了，即使换素材/换场景/换角度也不行。

### 铁律10：对标数据质量过滤

从 `data.json` → `benchmarking.videos` 或 `ranking.allVideos` 读对标原型时，必须检查每个原型的播放量：

| 播放量 | 判定 | 动作 |
|:------|:----|:-----|
| < 500 | ❌ **不可用** | 该视频自己都没跑过第一波推荐，不能作为对标原型 |
| 500-2000 | ⚠️ **低播放参考** | 可以借鉴结构但必须标注'该原型播放低'，不能单靠它决定类型方向 |
| >= 2000 | ✅ 可用 | 正常参考 |
| >= 5000 | ✅ 优质参考 | 推荐优先参考 |

**特别注意：** 如果对标库中该类型的所有参考视频播放均<2000 → 类型方向存疑，需要更多数据支撑才能选用该类型。

#### 铁律10检查结论模板

```
### 对标质量检查
- 选用的对标原型：[视频名] — 播放：[数字] — 判定：[可用/低播放参考/不可用]
- 该类型的对标库播放区间：[最低] ~ [最高]
- 结论：□ 可用 ✅ □ 存疑 ❌
```

### 铁律11：低播放视频分析

**规则：** 如果 `data.json` → `ranking.videos` 或 `postPublishTracking.videos` 中存在 播放<300 且 已发布>24h 的视频，写脚本前必须读其分析原因。

分析原因从以下维度判断：
- **封面问题**：是否没有人物/文字/反差
- **标题问题**：是否陈述句/太平淡
- **发布时间**：是否撞车或定时失败
- **限流**：是否被算法降权
- **内容问题**：是否方向偏了

分析结果可能存在 `data.json` → `analyses` 或 `reasoning` 字段中。写脚本前检查是否存在相关分析，如果存在则必须读并引用。

#### 铁律11检查结论模板

```
### 低播放视频分析引用
- 低播放视频：[标题] — [播放]
- 已有分析：□ 是 — 分析结论：[摘要] □ 否
- 本脚本是否有类似风险：□ 是（规避方式：___） □ 否
```
