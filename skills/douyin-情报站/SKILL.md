# douyin-情报站

## 职责范围

情报站不仅是调度器，**更是数据诊断中枢**。

- **调度任务：** 对标采集、创作者中心采集、每日简报、发后追踪（原始职责）
- **新增核心流程：** 账号诊断 → 方向建议（读全量数据，给出下一条视频的明确方向）

## 任务分派表

| 你要做什么 | 读哪个SKILL | 操作机器 |
|:----------|:-----------|:--------|
| 搜赛道爆款、采集对标数据 | `skills/douyin-对标采集/SKILL.md` | 🖥️ Windows |
| 采集自己账号的作品深度数据 | `skills/douyin-创作者中心/SKILL.md` | 🍎 Mac |
| 出每日简报、对比数据变化 | `skills/douyin-每日简报/SKILL.md` | 当前会话 |
| 发后追踪（采集数据+出简报） | 先创作者中心采集，再用每日简报出报告 | 🍎 Mac + 会话 |
| 账号诊断+方向建议 | **本文件（下面的诊断流程）** | 当前会话 |

## 数据枢纽

所有子技能读写同一个 `shared/data.json`，不存在多份数据副本。

```
对标采集 → 写入 benchmarking
创作者中心 → 写入 creatorCenter
每日简报 → 读取 benchmarking + creatorCenter + postPublishTracking
发后追踪 → 写入 postPublishTracking
账号诊断 → 读取全部字段 + materials/*.md + analyses
```

# 账号诊断流程

**调用方式：** 峰峰说"情报站，看看今天发什么"、"账号诊断"、"给个方向"

**这是一个纯分析流程**，不需要打开浏览器。只读 `shared/data.json` 和 `shared/materials/*.md`。

## 第一步：读全量数据

读取 data.json 的所有相关字段。**不要跳跃，逐字段读：**

1. `account` — 粉丝数、获赞、昵称
2. `creatorCenter.lastSnapshot.videos` — 最近作品播放量（按发布时间倒序）
3. `creatorCenter.depthData` — **重点：每条视频的完播率、2s跳出率、涨粉、粉丝播放占比**
4. `creatorCenter.followerAnalysis` — 千播涨粉率
5. `postPublishTracking.videos` — 发后追踪（含checkpoints、completionRate、exit2sRate、trafficSources、judgement）
6. `ranking.videos` — 播放量排序
7. `analyses.lowPlayback` — 低播放分析
8. `benchmarking` — 对标库（videos、competitors、visualAnalysis）
9. `materials` 及对应的 `materials/*.md` 文件 — 素材状态
10. `events` — 待处理事件

## 第二步：诊断六大维度（含数据对比）

**⚠️ 注意：** 每个维度不仅要读当前值，**必须对比历史值**。历史参照物包括：
- 上次诊断报告（看 memory/ 最近记录或 data.json 中的历史快照）
- 全账号平均值（计算最近所有有数据的视频取均值）
- 赛道健康值（对标库的典型值）

**对比后必须给出基于对比的建议，格式如下：**
- 当前值 vs 历史值：x → y（涨/跌）
- 是否异常：□ 正常 □ 预警 □ 恶化
- 基于对比的建议：根据变化趋势给出下一步动作

### 维度A：播放健康度

**读数据源：** creatorCenter.lastSnapshot.videos + ranking.videos + postPublishTracking

逐条分析最近5条视频：
- 每条播放量是多少？同比上一条涨了还是跌了？
- 播放<500的视频有几个？各是什么原因？
- 是否有多条连续低播放（存在内容危机）？
- 是否有爆款趋势（D+1/D+3 播放超过之前均值的200%）？

**输出：**
- 播放状态：正常 / 下滑 / 异常（注明原因如定时故障）
- 倒数3条播放趋势：↑ / ↓ / =
- 与上次诊断对比：总播放从xxx到xxx，变化xx%
- 建议：□ 继续当前方向 □ 暂停审核 □ 换方向（注明理由）

### 维度B：完播与跳出

**读数据源：** creatorCenter.depthData + postPublishTracking

逐条分析有深度数据的视频：
- 完播率最高/最低的各是哪条？原因是什么？
- 2s跳出率最高/最低的各是哪条？
- 整体平均值是多少？

**输出：**
- 完播：xx%（趋势↑↓）
- 2s跳出：xx%（趋势↑↓）
- 与上次诊断对比：完播从xx→xx（±x%），2s跳出从xx→xx（±x%）
- 诊断：□ 正常 □ 跳出偏高 □ 完播偏低
- 建议：□ 无变化，继续验证 □ 完播下降需改结构 □ 跳出上升需改开头

### 维度C：涨粉转化率

**读数据源：** creatorCenter.followerAnalysis + postPublishTracking

- 千播涨粉率 = 涨粉总数 / 总播放 × 1000
- 对比健康值（2-5/千播）
- 最近3条视频各自涨了多少粉？
- 结尾有没有身份锚点？
- 连续零涨粉天数

**输出：**
- 千播涨粉：x.xx（健康值2-5）
- 与上次诊断对比：从x.xx到x.xx
- 最近N条涨粉情况：（哪条涨了几个，哪条0）
- 风险：□ 正常 □ 极低 □ 已连续多日零涨粉
- 建议：□ 继续验证转化率 □ 结尾必须加身份锚点 □ 改互动引导

### 维度D：素材使用状态

**读数据源：** materials + materials/*.md + shared/scripts/ 目录下最近已生成的脚本文件

- 当前有哪些P0素材未使用？
- 哪些素材刚用过且数据差（应禁用1周）？
- 哪些素材scriptReady但没发？
- **shared/scripts/ 中已有脚本用了哪些素材和角度？**

**素材禁用规则：**
- 上周有用过且该视频播放<500的素材 → 禁用（至少本周不再用）
- 连续2条用同素材（不管播多少）→ 禁用1周
- 同素材在不同角度下数据差异大的 → 可保留但需标注数据
- **shared/scripts/ 中已生成脚本占用的素材/角度 → 如果脚本已发布，禁止推荐同一素材的同一角度**

**输出：**
- 可用素材池：[素材ID列表]
- 禁用素材池：[素材ID列表]
- 与上次诊断对比：上周推荐了xxx，实际使用了xxx，效果如何
- 推荐优先使用的素材：[素材ID + 理由] + 基于数据的原因

### 维度E：对标与赛道

**读数据源：** benchmarking

- 最近赛道高播放类型是什么？（对比型/他人视角型/感慨型/冲突型）
- 对标库中TOP3是什么模板？
- 与自身账号人设的匹配度？

**输出：**
- 赛道趋势：xx类型播放最高
- 与上次诊断对比：赛道高播放类型有无变化
- 可参考对标：xxx（播放x万，结构xxx）

### 维度G：涨粉转化诊断（NEW ⛔ 强制）

**严重程度：** ⚠️ 最高优先级。峰峰账号播放量在提升（老好人1346播+34.71%完播），但**涨粉趋近于0**。完播率最高（34.71%）的视频涨粉1个都没有。这是当前账号增长最大的瓶颈。

**读数据源：** creatorCenter.depthData + postPublishTracking.videos + creatorCenter.followerAnalysis

#### 诊断步骤

**1. 计算每条视频的涨粉转化率**

对最近N条有数据的视频（播放≥500），逐条计算：

```
千播涨粉 = 涨粉数 / 播放数 × 1000
健康值：2-5/千播  当前值：~0-2/千播
```

标出哪些视频千播涨粉=0，哪些>0。

**2. 归因分析：0涨粉视频的共性特征**

对千播涨粉=0的视频，检查以下维度：

| 检查项 | 说明 |
|:-------|:-----|
| 结尾是否有身份锚点 | 观众看完是否知道"这是谁、关注他能看到什么" |
| 结尾接话空间是否≥3种 | 如果只2种（是/否选择题）=接话空间小，互动少→涨粉低 |
| 评论区互动数量 | 互动多的视频（如刚跑了两天33评）即使完播低也能涨粉 |
| 粉丝播放占比（fanPlayRatio） | <2%说明几乎没有新粉转换 |
| 结尾是否属于格言型/抽象大词型/总结式 | 这三种结尾看完就走，零涨粉 |
| 视频类型 | 感慨型回看数据、对比型/他人视角是否比单纯的内心独白型涨粉好 |

**3. 数据对比输出格式**

```
┌──────────┬────────┬──────┬──────────┬──────────┬──────────┐
│   视频   │ 播放   │ 涨粉 │ 千播涨粉 │ 结尾类型 │ 互动量   │
├──────────┼────────┼──────┼──────────┼──────────┼──────────┤
│ 老好人   │ 1,346  │ 0    │ 0.00     │ 自嘲式   │ 1评      │
│ 跑了7年  │ 3,306  │ 3    │ 0.91     │ 两面选择 │ 5评      │
│ 刚跑两天 │ 8,271  │ 1    │ 0.12     │ 冲突式   │ 33评     │
│ 凭啥不如 │ 2,646  │ 0    │ 0.00     │ 自嘲式   │ 2评      │
└──────────┴────────┴──────┴──────────┴──────────┴──────────┘
```

**4. 根因提炼模板**

每条0涨粉视频必须给出具体原因，格式：

```
- 视频标题（XXX播/XX涨粉）：原因1（具体） | 原因2（具体）
```

**5. 改进建议输出**

对每条即将产出的脚本，给出具体的涨粉改进建议，不是笼统的"加身份锚点"，而是写死到脚本里：

```
✅ 好的建议：这个脚本结尾应该加一句"跑了七年的人，改不了了" — 把自己干的事说出来
❌ 坏的建议：结尾加身份锚点 — 太笼统，脚本生成器不知道怎么落地
```

## 第三步：综合判断与方向建议

### 脚本生成器参考

列出当前账号最需要解决的1-3个问题，每个问题附上数据证据。

**问题排序按严重程度，涨粉转化问题强制排第一。**

例如：

4. 素材池枯竭/某素材过度使用（证据：xxx）

### 下一条视频的方向建议

- **为什么选这个：** （数据依据，例如"对标库里他人视角F平均播放最高"或"最近没有用过对比型"）

- **风险提示：** （例如"这个素材的某角度之前用过但数据不好"）

**禁用的方向（如果有）：**

## 安静模式

- 数据正常时：输出诊断报告但不主动推，等峰峰下次聊天时告知
- 数据异常时（连续低播放/零涨粉/素材枯竭）：直接输出给峰峰看
