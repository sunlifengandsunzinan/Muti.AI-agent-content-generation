## 强制自检清单

### □ 1. 画面类型选择检查
- 脚本类型是自省/感悟/吐槽类？→ 必须用B类（固定机位说）
- 脚本类型是纯跑步经历分享？→ 可尝试A类（跟拍跑动）或C类（走路边走边说）
- 如果选了B类以外的类型 → 理由：_____________
- 理由充分？ □ 是 ✅ □ 没有硬数据支撑 ❌

### □ 2. 取优点检查（对标播放前5）
- [ ] 找到脚本的情绪波动点在哪几秒
- [ ] 情绪表情类型与播放前5一致（真诚自嘲/无奈/想通了）
- [ ] 镜头是胸部以上近景（播放前5全部是这个景别）
- [ ] 没有演过头的风险

### □ 3. 避坑检查（对标播放后5）
- [ ] 画面和配音情绪一致（避免画面灿烂但配音沉重）
- [ ] 开头第一帧有吸引力（避免黑屏/空白/转头动作）
- [ ] 表情自然（避免太苦/太用力/像在演戏）

### □ 4. 单人可操作检查
- [ ] 手机放路边固定 / 前摄自拍 / 手持空镜
- [ ] 不需要第二个人协助
- [ ] 不需要额外设备

### □ 5. 口型同步标注
- [ ] 每个镜头标注了"需要/不需要"口型同步
- [ ] 大多数镜头标"不需要"（后期配音前提）

### □ 6. 画面多样化检查 ✅ 新增
- [ ] 本次画面类型与最近2条不重复（避免同类化）
- [ ] 最近2条画面类型：_____________ / _____________
- [ ] 本次选：_____________
- [ ] 如果重复 → 挑一个不同类型的方案

### □ 7. 涨粉镜头检查 ✅ 新增
- [ ] 结尾画面有没有拍正面/正脸？（涨粉要靠人脸记忆）
- [ ] 封面截帧画面在拍摄方案中标注了哪一段
- [ ] 有没有至少1个镜头让观众记住'这张脸'

⛔ 全部通过 ✅ 才能输出
