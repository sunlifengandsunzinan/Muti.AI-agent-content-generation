# 2026-07-15 周三 — 情报站架构整改+今日采集

## 10:00 架构整改
### 发现问题
- cron任务配置为 isolated session，全部超时失败
  - 📡 抖音数据监控-每日采集（8:00）：连续14次错误
  - 📡 情报站-创作者中心数据拉取（8:30）：连续6次错误
  - 📡 情报站-每日简报（9:00）：连续6次错误
- 根因：isolated session timeout 门槛太低（600s），模型操作Mac浏览器时间不够

### 整改方案：双保险机制
**核心思路：cron只发轻量systemEvent闹钟到主会话，主会话通过心跳/聊天状态来处理任务**

#### 改动1：三个cron任务改为主会话闹钟
| 原任务 | 改为 | sessionTarget |
|:------|:----|:------------:|
| 抖音数据监控-每日采集（8:00） | 📡 闹钟-账号基础数据采集 | main |
| 情报站-创作者中心数据拉取（8:30） | 📡 闹钟-创作者中心数据 | main |
| 情报站-每日简报（9:00） | 📡 闹钟-每日简报 | main |

所有任务启用 `sessionTarget="main"` + `payload.kind="systemEvent"`，只发一句话提醒。

#### 改动2：HEARTBEAT.md 心跳检查
写入了心跳轮询任务清单：检查events.json、检查今日是否完成采集、检查发后追踪、检查新视频、检查异常数据。

#### 改动3：现有闹钟超时拉到900秒
预留兜底：万一isolated直接跑，15分钟够用了。

### 子Agent vs isolated session
- subagent（spawn-child）：跑Mac浏览器OK，上次13小时前那次91k token正常完成
- isolated session cron：连续超时
- **结论：如果流程需要跨机器操作，必须走主会话→subagent的链路，不能走isolated session**

## 12:45 实战教训：强制自检清单
### 问题
脚本生成器产出015脚本「以前上三楼喘半天」质量差：
1. 前10字是陈述句（"以前上三楼喘半天"）
2. 自问自答堵观众嘴（"你说这7年到底干了啥？也没干啥"）
3. 格言型大道理（"就是把时间花在自己身上了"）

### 根因
子Agent虽然读了SKILL.md的规则，但没有**逐条执行检查**。它自己找了个"画面感入戏"的借口就过了自检。

### 整改
**所有内容产出型Agent + 强制自检清单**，写完后逐条打勾，不通过不出方案。

#### 脚本生成器 — 7项自检
1. 前10字钩子类型（反问/悬念/冲突/数字/情绪词）
2. 结尾类型（没整明白式/你发现没式/两面选择式/自嘲接话式）
3. 球给观众（写一句模拟回复验证）
4. 观点不重复（vs最近1条已发）
5. 口语化（逐句读一遍）
6. 时长字数（8-10秒/55-70字）
7. 敏感词（无病/死/惨/疯了/完了）

#### 标题封面官 — 4项自检
1. 标题-封面互补
2. 封面风格规范（居左/居中、白字黑边、无头像）
3. 标签上限5个、主标签跑步+情绪
4. 标题留悬念、封面揭答案

#### 画面设计师 — 5项自检
1. 画面类型选对（自省类→B类固定机位）
2. 取优点（对标播放前5情绪波动点、近景景别）
3. 避坑（开头帧吸引力、情绪匹配）
4. 单人可操作
5. 口型同步标注

### 效果验证（13:24）
脚本生成器重写016「凑整强迫症」，7项自检全部通过 ✅
- 前10字：反问「跑个步你非得凑个整吗」
- 结尾：你发现没式「强迫症这事儿，到哪都跑不掉」
- 口头化：凑个整/少一米都不舒服/买菜也一样
- 56字/9秒
- 观点不重复（自我主义执念）
